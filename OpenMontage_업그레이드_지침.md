## A. 즉시 적용 — OpenMontage `CLAUDE.md`에 추가할 지침
> 코딩 거의 없음. Claude Code에 "이 내용을 CLAUDE.md에 추가해줘" 하면 됨.
> 원칙: **규칙은 적고 명확하게**(REF-8) — 아래는 "꼭 지킬 것"만.

### A-1. 한국어 나레이션
- 대본의 **숫자는 한글로** 적는다. (5분→오 분, 3가지→세 가지, 2리터→두 리터) ← 발음 오류 방지(테스트에서 "오사분" 발생)
- 외래어·영문은 한글 발음으로 풀어 적는다. (AI→에이아이 등, 필요 시)
- 기본 음성: **ElevenLabs / voice_id `4JJwo477JUAx3HV0T7n7`(yohan koo) / model `eleven_multilingual_v2` / voice_settings stability 0.5·similarity 0.85·style 0·speaker_boost on**. (v3는 쓰지 말 것 — API 불안정)

### A-2. 한국어 자막
- 자막은 **문장·구(節) 단위로** 끊는다. **다음 문장의 첫 단어를 이전 장면에 남기지 않는다.** (테스트에서 "내일"이 앞 장면 끝에 댕글링)
- 폰트: **Pretendard 또는 Noto Sans KR** 고정 (글자 깨짐 방지). 시스템에 `fonts-noto-cjk` 설치됨.
- 한 줄 글자 수는 **12~16자 내외**, 화면 **하단 1/3**, 핵심 단어는 **강조색**(REF-4).
- **나레이션 대본 = 자막의 원본(source of truth).** 자막 텍스트는 대본과 일치시킨다(받아쓰기 오류 시 대본으로 교정 — REF-13 SRT 교정).
- 세로(9:16) 영상은 **하단 IG 세이프존**(좋아요·댓글 버튼 영역)에 자막이 안 걸리게(REF-16).

### A-3. 그림체(이미지 스타일) — 카테고리별 스타일 템플릿으로 관리
- **그림체는 CLAUDE.md에 하나로 고정하지 않는다.** 대신 **카테고리별 스타일 템플릿**(교육/패션/브이로그…)에서 지정한다.
- 영상 제작 시 "이 영상은 [교육] 템플릿으로" 라고 하면, 그 템플릿의 그림체·색감·자막·모션을 적용.
- **단, 한 영상 안에서는 그림체를 통일**한다. 가능하면 레퍼런스 이미지 1장 기준(REF-2·3 Elements).
- 스타일 템플릿 위치: `templates/` 폴더 (뼈대: `_템플릿_뼈대.md`).

### A-4. 자체 검수(렌더 후, 사용자에게 보여주기 전) — REF-13·15
- 자막: 깨짐/잘림/댕글링 단어 없는지, 싱크 맞는지
- 음성: 숫자·발음 어색한 곳 없는지(대본과 대조)
- 컷 경계: 어색한 점프/오디오 팝 없는지
- 문제 있으면 고쳐서 다시 렌더(최대 3회) 후 제시

### A-5. 작업 원칙(메타) — REF-8·13
- **많은 규칙보다 적은 원칙.** 위 규칙 외엔 AI의 창작 자유.
- **100% 자동화보다 효율화** — 어색하면 멈추고 사용자 확인.

### A-6. 결과물 저장 위치 (표준)
- **완성된 최종 영상은 반드시 아래 폴더로 복사**한다:
  - Windows: `D:\peaceworks\영상편집자동화\영상 편집 자동화\완성영상\`
  - WSL: `/mnt/d/peaceworks/영상편집자동화/영상 편집 자동화/완성영상/`
- 파일명: `YYYYMMDD_주제_카테고리.mp4` (예: `20260624_아침스트레칭_교육.mp4`)
- OpenMontage 내부 작업본(projects/…)은 그대로 두고, **최종본만 위 폴더에 복사**.

### A-7. 제작 워크플로우 — 스크립트 우선 + 캡컷 드래프트 핸드오프 (★가장 중요)
- **1단계: 스크립트(대본)만 먼저 생성해 사용자에게 보여준다.** 이미지·음성·음악을 만들기 전에, 사용자가 **검토·수정·승인**할 때까지 멈춘다. (스크립트가 source of truth — REF-11·13)
- **2단계: 승인된 스크립트로만** 이미지·음악·자막을 생성한다.
- **3단계: 최종 렌더(완성본) 대신 "편집 가능한 형태"로 핸드오프** → 사용자가 캡컷에서 후반 편집한다.
  - 이상적: **캡컷 드래프트(프로젝트 파일) 내보내기** (※ 현재 OpenMontage엔 없음 — 개발 필요, 로드맵 C-5)
  - 당장(중간 방식): **재료를 한 폴더에 정리해 내보내기** — 이미지들 / TTS 음성 / 음악 / 자막(SRT) 따로 → 사용자가 캡컷에서 직접 조립
- 이유: 스크립트에서 오류를 먼저 잡고, 나머지는 후반에서 사용자가 통제. 잘못 만들어도 재생성 비용·수정 최소화.
- **나레이션(TTS) 처리 방침(확정)**: yohan koo로 TTS를 **생성하되 영상에 굽지 말고 별도 음성 트랙**으로 핸드오프 → 캡컷에서 수정·교체·타이밍 조정 가능.
- **핸드오프 묶음(중간 방식)**: 한 폴더에 ① 이미지/영상 클립들 ② TTS 음성(별도 파일) ③ 음악(별도 파일) ④ 자막 SRT → 캡컷에서 불러와 마무리. (최종 mp4 자동 렌더 X — 사용자가 후반 편집)

### A-8. 영상생성(image-to-video) 비용 전략 (기본값 — 매번 지시 불요)
- **핵심 컷 2~3개만 image-to-video(fal.ai Kling)** 로 진짜 모션을 준다.
- **나머지 컷은 Remotion 애니메이션**(줌·켄번즈·패럴랙스)으로 처리해 비용을 낮춘다.
- 가능하면 **start/end 프레임**으로 모션을 통제한다(REF-2·3).
- 사용자가 매번 지시하지 않아도 **항상 이 방식이 기본**. 다르게 원할 때만 사용자가 지정.
- (영상생성 프로바이더: fal.ai = FAL_KEY. 부족하면 Higgsfield MCP 추가.)

### A-9. 이미지 모델·텍스트·실사 규칙
- **이미지 모델 = gpt-image-2 사용** (gpt-image-1은 한글 깨짐 심함 → 교체). 비용 더 들어도 한글·실사 품질 우선.
- **gpt-image-2는 한글/짧은 텍스트를 비교적 잘 렌더** → 이미지 안 짧은 라벨은 OK. 단 **중요한 한국어 자막은 여전히 Remotion/캡컷 자막 레이어**로 얹는 게 안전(검수·수정·일관성).
- **실사 이미지는 `templates/realistic-image-prompt-guideline.md` 를 따른다** (실제 카메라/조명/질감/결함 + 금지사항). 장면 한 줄 → 에이전트가 그 지침대로 상세 영어 프롬프트로 확장 → gpt-image-2 생성.
- **업로드용은 "AI-generated content" 워터마크**(프롬프트 + 후처리 + 플랫폼 표시) — 유튜브 AI 공개 정책 대응.
- 스타일(실사/일러스트)은 비용 무관. 비용은 해상도·품질티어·모델·영상생성 클립 수로 결정.

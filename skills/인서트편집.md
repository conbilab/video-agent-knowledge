# 인서트편집

## 이 기능이 하는 일 (정확히)

- **입력**: 내가 이미 만든 "완성 영상"(내 촬영·설명본) 하나.
- 나는 자료화면(B롤)을 **안 갖고 있다**. 없는 자료를 **AI가 새로 만들어야 한다.**
- 영상의 문맥 + 자막(대사)을 파악 → 어느 구간에 어떤 시각 자료가 있으면
  이해가 쉬운지 판단한다.
- 그 자료를 **AI 이미지 생성 → image-to-video**로 영상 전환한다.
- 그 영상을 **원본 영상 위 인서트 컷(오버레이 트랙)**으로 해당 구간에 올린다.
- 최종은 내가 더 손볼 수 있게 **캡컷(CapCut) 드래프트**로 출력한다.

## 파이프라인

1. **전사**: faster-whisper 로 자막·타임코드(대사 + 시작/끝 시간) 추출.
2. **인서트 지점·내용 판단**: 문맥과 자막을 근거로, "여기에 이런 시각 자료가
   있으면 이해가 쉬워진다"는 구간만 고른다. 각 지점마다 (타임코드, 무엇을 보여줄지,
   정지/영상 여부)를 정한다.
3. **이미지 생성**: `docs/knowledge/templates/포맷결정_지침.md` 로 포맷(실사/일러스트/
   영상/정지)을 결정하고, 실사면 `docs/knowledge/templates/realistic-image-prompt-guideline.md`
   를 따라 프롬프트를 쓴다.
4. **image-to-video**: 핵심 지점만 `docs/knowledge/templates/video-model-selection-guideline.md`
   를 따라 모델·비용을 골라 영상으로 전환한다. (기본 Kling)
5. **배치**: CapCut Agent(HTTP, 포트 8765)로 **오버레이 트랙**의 해당 타임코드에
   각 인서트를 배치한다.
6. **출력**: 캡컷 드래프트를 출력한다(내가 이어서 손볼 수 있게).

## 규칙

- **꼭 필요한 곳만.** 이해에 도움되는 지점에만 넣는다. 장식 남발 금지. 인서트 길이 2~4초.
- **비용**: 핵심만 image-to-video, 나머지는 정지 이미지 + 살짝 줌(켄번즈).
- **한글**: 이미지에 깨지는 한글 넣지 말 것(자막은 레이어). AI 생성물엔
  'AI-generated content' 워터마크.
- **CapCut Agent는 상용 제품**이다 → HTTP 느슨한 결합만 사용한다. 코드 병합 금지,
  배치 지시만 넘긴다.
- **CapCut Agent 무응답(포트 8765)**이면: 다운받은 자료가 유실되지 않게
  **생성한 에셋 파일 + 배치 계획(타임코드 표)**만 출력하고, Agent가 응답 없음을 알린다.

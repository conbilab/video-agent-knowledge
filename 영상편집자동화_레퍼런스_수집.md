# 영상편집 자동화 — 레퍼런스 수집 & 에이전트 설계 노트

> **목표**: 유튜브 등에서 모은 영상편집 자동화 정보를 축적해, 최종적으로 "가장 좋은 영상편집 자동화 에이전트"를 만든다.
> **현재 단계**: 정보 수집 (계속 추가 예정). 새 영상이 들어오면 이 문서에 항목을 덧붙인다.
> 최종 업데이트: 2026-06-19

---

## 0. 큰 그림 — 두 갈래로 나뉜다

| 갈래 | 하는 일 | 대표 도구 | 비용 | 우리 목표와의 관계 |
|---|---|---|---|---|
| **A. 실제 영상 편집 자동화** | 내가 찍은 영상을 컷·자막·효과로 다듬기 | Claude Code + video-use / Whisper / FFmpeg | 거의 무료 (받아쓰기만 일부 유료) | ⭐ **핵심 목표** |
| **B. AI 영상 생성** | 없는 영상을 AI로 창작 (가상 인물·광고·시네마틱) | Higgsfield 등 | 유료 구독 | 보조 (필요 시 A와 연결) |

→ "가장 좋은 에이전트"는 **A를 중심으로** 만들고, B는 소스 생성용으로 옵션 연결.

### [REF-7] 큐레이터 단비 — "프로그램 말고 스킬+CLI" 자동화 철학 ⭐⭐(에이전트 설계 핵심)
- **갈래**: A(편집 자동화) + **설계 철학**(우리 에이전트 구조를 좌우하는 핵심 자료)
- **핵심 주장**: 버튼 많은 **프로그램(앱) ✕ → 스킬 + CLI ○**
  - 프로그램은 **How를 고정** → 고칠 때마다 수정·테스트·재배포 반복("일을 위한 일")
  - **스킬** = AI가 읽는 **작업 설명서(텍스트)**, 로직+설정만 담음 → 흐름 바꾸기 쉬움
  - **CLI/터미널** = AI가 컴퓨터에서 직접 일하는 통로 (가볍고 빠름)
  - **MCP**는 토큰 많이 쓰고 다소 느리고 설정 필요 → 혼자 쓰는 자동화엔 **스킬+CLI가 더 빠를 때 많음**
- **영상편집**: 롱폼 1개 → **숏폼 여러 개 자동 추출** (12분 영상 → 숏폼 5개, 사람 편집 0)
- **다른 스킬 예시**: AI 이미지 갤러리(생성·변환·업로드·SEO·링크), 색칠 도안, TTS(슈퍼톤), 네이버 포스팅 — 전부 "말 한마디 → 스킬 자동 실행"
- **도구**: Cursor(메인), Codex, Claude, Antigravity, opencode — 다 깔아보고 본인에 맞는 것 선택
- **고급 운영**: 에이전트마다 스킬 지정 → 24/365 자동, 텔레그램으로 원격 지시
- **실전 팁**: 거창한 프로그램 X, **반복작업 하나를 스킬로 쪼개기** / 스킬은 한 번 만들고 끝이 아니라 **계속 고쳐 쓰는 자산**
- **교보재 링크(미확인)**: 영상편집 자동화 로직(Whisper+FFmpeg+Remotion) — https://min-inter.co.kr/wiki/video-editing-automation-whisper-ffmpeg-remotion/
- **에이전트에 쓸 점(중요)**: 우리 "최고 에이전트"는 **앱/프로그램이 아니라 "스킬 묶음 + CLI"로 설계**해야 한다. → 이미 깐 video-use/claude-video/HyperFrames가 정확히 이 방식. **롱폼→숏폼 다중 추출 스킬**을 후보로 추가.

### [REF-8] (한국 크리에이터) — Claude Code + Remotion + 5개 원칙 ⭐⭐⭐(시스템 설계 인사이트, 최고 가치)
- **갈래**: A/B 경계(코드로 모션그래픽 영상 자동 생성) + **AI 시스템 설계 철학**
- **도구**: **Remotion** = "코드를 적으면 영상이 되는" 도구(설계도 비유). Claude Code가 코드를 짜고 Remotion이 렌더. (GitHub 39k★, 월 90만 설치, 문서 700p)
- **셋업**: ① Claude Code ② **Remotion best practices**(잘 쓰는 법 레시피 모음) ③ 스킬
- **3가지 실패(핵심)**:
  1. **스킬만 덕지덕지** → 검은 배경 흰 글자(PPT 수준). **교훈: 원칙/가이드 없이 스킬만 붙이면 실패**
  2. **8만 장 프레임 벤치마킹**(잘 만든 채널 영상을 1프레임씩 8만 장 외우게 함) → 그 채널 스타일은 따라하나 **새 대본엔 멘붕**(기출 외운 학생). **교훈: 레퍼런스는 영감용이지 복사용 아니다. 정답 외우게 하면 응용력 0**
  3. **규칙 1800줄** → AI가 규칙 따르기에만 급급, 규칙끼리 충돌, 아무것도 못 만듦. **교훈: 규칙 많으면 AI가 규칙 따르기 바쁘고, 원칙 적으면 AI가 생각하기 시작한다**
- **전환점 질문**: "나는 그림을 한눈에 보는데 AI는 왜 못 보지?" → AI: "나는 그림을 직감으로 느끼는 게 아니라 글 지시를 한 줄씩 읽으며 코드를 짠다" → **AI는 사람과 다르게 작동**
- **해결: 1800줄 → 5개 원칙**. 원칙은 둘로: ① **꼭 지킬 규칙**(채널 스타일·코드 문법 — 어기면 깨짐) ② **잘하는 일엔 자유**(장면 연출·감정 표현 — AI가 더 많은 변형). **규칙을 줄였더니 퀄리티가 올라감**
- **구성법**: AI에게 직접 물음 — "너한테 어떤 규칙 주면 가장 잘 만들어? 어디까지 자유 줘야 해?" → 제안→확인→반복
- **핵심 인사이트 4**: ① 많은 규칙보다 **적은 원칙** ② AI는 사람과 다르게 본다 ③ 실패가 가장 빠른 길 ④ "왜 안 돼?"가 아니라 **"이게 원래 어떻게 작동하지?"** 를 물어라

## 2. 공통 패턴 (여러 자료에서 반복되는 것)

1. **입력 폴더 → 출력 폴더** 단순 구조
2. **무음/군더더기 컷 자동 제거**
3. **받아쓰기 기반 자막** (Whisper 무료 vs ElevenLabs Scribe 유료·정확)
4. **자막 커스터마이징**: 위치·글자수·폰트·강조색, 영상에 직접 새김
5. **효과음·이미지·애니메이션**으로 풍성하게
6. **포맷/지침으로 고정 → 한 마디로 반복 자동화**
7. **1분 짧은 영상으로 반복 튜닝 → 만족 시 지침 반영**
8. 프롬프트/판단은 **에이전트에게 위임**
9. **"앱/프로그램" 대신 "스킬 + CLI"** 로 설계 (수정·확장 쉬움) ← REF-7 핵심 원칙
10. **많은 규칙(rules)보다 적은 원칙(principles)** + 꼭 지킬 하드룰 ← REF-8 핵심 (video-use 설계와 동일)
11. **레퍼런스는 영감용**이지 복사·암기용 아님 ← REF-8 (REF-1 보정)
12. **AI는 사람과 다르게 작동** → 사람 시키듯 X, AI 방식대로 지시 ("왜 안 돼?" 대신 "원래 어떻게 작동?")
13. **100% 자동화보다 "효율화"** — 인간적인 호흡 유지, 반자동 옵션 ← REF-13
14. **차이를 만드는 3가지: 명확한 지시 + 빠른 반복 + 직접 해보기** ← REF-13(REF-2·9·11과 동일 결)

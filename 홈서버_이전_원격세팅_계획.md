# 홈서버 이전 + 어디서든 영상 제작 — 세팅 계획

> 목표: 영상 자동화를 **홈서버**로 옮기고, **폰에서 어디서든** "이런 영상 만들어줘" → 기획안 받고 → OK → 자동 제작 → 결과 받기.

---

## 0. 결론 — "open claw" = OpenClaw (딱 맞는 도구)
- **OpenClaw** (github.com/openclaw/openclaw): 폰 챗앱(텔레그램·iMessage 등) ↔ AI 에이전트를 잇는 **자가호스팅 다리**. 무료.
- **openclaw-code-agent** 플러그인: OpenClaw 밑에서 **Claude Code 실행** + **기획안 승인(Approve/Revise/Reject)** 흐름 내장. → 우리가 원한 워크플로 그대로.
- 그래서 **텔레그램 봇을 따로 만들 필요 없음.** OpenClaw가 그 역할.

### ⚠️ 이름 구분 (중요)
- **OpenClaw** = 폰 ↔ 에이전트 **다리** (연결·기획승인)
- **OpenMontage** = 실제 **영상 제작기**
- 둘은 다른 거, 같이 일함.

---

## 1. 전체 구성
```
폰/노트북 (어디서든)
  │  Phase1: SSH 터미널   /   Phase2: 텔레그램 메시지
  ▼
Tailscale (안전 연결, 무료)
  ▼
홈서버 (우분투 + 도커)  ← 이미 준비됨 ✓
  ├ OpenClaw       폰↔에이전트 다리 (기획안→승인)
  ├ Claude Code    두뇌 = 기획·지휘  ← 우리 지식폴더(CLAUDE.md·지침·템플릿)
  └ OpenMontage    실제 제작 (gpt-image / fal / ElevenLabs)
  ▼
완성 영상 → 폰으로
```

---

## 2. 옮기는 것 (대부분 그냥 복사)
| 대상 | 방법 |
|---|---|
| OpenMontage (프로그램) | 서버에서 `git clone` 새로 받기 (제일 깔끔) |
| `영상편집자동화` 지식폴더 (지침·템플릿·docs·CLAUDE.md 커스텀) | 복사(rsync) 또는 개인 git repo로 |
| 메모리 파일 | 복사 |
| `.env` (API 키) | **복사 말고 서버에서 다시 입력** (보안) |

---

## 3. 진행 순서 — 터미널 먼저 → 텔레그램 (선택대로)

### Phase 1 — 터미널 원격 (먼저, 간단)
1. **Tailscale** 설치 → 폰/노트북 ↔ 서버 안전 연결
2. **파일 이전** (위 표대로). `.env` 키는 서버에서 재입력
3. 서버에 **Claude Code + OpenMontage 의존성** 설치 (Python 3.12, Node, ffmpeg 등)
4. SSH로 접속 → `claude` 실행 → **3pH 영상 원격 제작 테스트** (되면 원격 파이프라인 검증 끝)

### Phase 2 — OpenClaw + 텔레그램 (그다음)
1. 서버에 **OpenClaw 설치** (도커 있으니 도커로)
2. **openclaw-code-agent 플러그인** 연결 (Claude Code + 기획 승인 버튼)
3. **텔레그램 봇** 연결
4. 폰에서 "이런 영상 만들어줘" → 기획안 → 승인 → OpenMontage 제작 → 결과 폰으로

### Phase 3 — 기획 강화 (지식 추가)
- **기획 워크플로 문서** 추가: 아이디어 → 앵글/후킹 → 대본 → 장면별 포맷(포맷결정_지침) → (원하면) 발행 일정
- OpenClaw 승인흐름(UX) + 이 문서(실제 기획력) = 두 층으로 기획 강화
